### Day 09 – Web Accessibility & Inclusive User Experience

On **Day 9** of the **50 Days Dev Web Development Challenge**, I focused on making my website more accessible by implementing **WCAG (Web Content Accessibility Guidelines)** principles. I improved the user experience for people who navigate using keyboards or assistive technologies by adding accessibility-focused features instead of visual enhancements.

I implemented a **"Skip to Main Content"** link to help keyboard users bypass the navigation menu, improved image accessibility by writing meaningful **alt text**, and added `:focus-visible` styles to provide a clear focus indicator during keyboard navigation. These improvements ensure that users can easily understand and interact with the website regardless of how they access it.

### What I Learned

* Understanding the importance of Web Accessibility (WCAG).
* Implementing a **Skip to Main Content** link for keyboard users.
* Writing meaningful **alt** text for images.
* Using `:focus-visible` to improve keyboard navigation.
* Creating a better user experience for users of assistive technologies.
* Auditing website accessibility using **Google Lighthouse**.
* Identifying and fixing accessibility issues to achieve a **perfect accessibility score**.

### Technologies Used
* HTML5
* CSS3
* Semantic HTML
* ARIA & Accessibility Best Practices
* Google Lighthouse

### Outcome

Successfully improved the accessibility of my community website by implementing keyboard-friendly navigation, descriptive image alternatives, and visible focus indicators. The website now provides a more inclusive user experience while achieving a **100/100 Accessibility Score** in **Google Lighthouse**.
