### Day 06 – Responsive Web Design with CSS Media Queries

On **Day 6** of the **50 Days Dev Web Development Challenge**, I learned how to make my website responsive using **CSS Media Queries**. I transformed the Synexus Official website into a layout that automatically adapts to different screen sizes, ensuring a better user experience on desktops, tablets, and mobile devices.

I implemented responsive breakpoints using `@media (max-width: 900px)` and `@media (max-width: 600px)` to reorganize the website layout based on the viewport width. The Initiatives Gallery was updated from a three-column grid on desktop to two columns on tablets and a single-column layout on mobile devices using `grid-template-columns`. I also modified the navigation bar to stack vertically on smaller screens, preventing overlapping and improving usability.

As a bonus challenge, I explored **Fluid Typography** by using the `clamp()` function for the Hero section heading, allowing the text to scale smoothly across different screen sizes. I also enhanced the website's appearance by refining the Hero section with a modern gradient background, a styled call-to-action button, and improved spacing to create a cleaner and more professional user interface.

### What I Learned

* Understanding Responsive Web Design principles.
* Using CSS Media Queries to create layouts for different screen sizes.
* Creating responsive breakpoints with `max-width`.
* Making CSS Grid layouts adapt from three columns to two and one column.
* Building mobile-friendly navigation layouts using Flexbox.
* Understanding Desktop-First responsive design.
* Implementing Fluid Typography using the `clamp()` function.
* Testing responsive layouts using Chrome Developer Tools.
