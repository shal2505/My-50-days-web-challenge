### Day 07 – Membership Application Form & Interactive UI

On **Day 7** of the **50 Days Dev Web Development Challenge**, I learned how to build and
style an interactive **Membership Application Form** using **HTML5** and **CSS3**. I created
a structured form using semantic HTML elements and linked each label to its corresponding 
input using the `for` and `id` attributes to improve accessibility and user experience. I 
customized the form by replacing the browser's default styles with a clean, modern design 
using custom padding, borders, rounded corners, and responsive spacing. I also implemented 
interactive CSS pseudo-classes such as `:focus`, `:hover`, and `:disabled` to provide visual
feedback when users interact with the form elements.As a bonus challenge, I explored **HTML5
Form Validation** by using the `required` attribute along with the `:valid` and `:invalid`
pseudo-classes to display real-time validation feedback. This helped me understand how to 
create accessible, responsive, and user-friendly forms with a professional look and feel.

### What I Learned

* Building accessible forms using HTML5.
* Linking labels and inputs using `for` and `id`.
* Styling form elements with CSS.
* Using `:focus`, `:hover`, and `:disabled` pseudo-classes.
* Implementing HTML5 validation with `required`, `:valid`, and `:invalid`.
* Creating clean, responsive, and user-friendly form interfaces.

### Technologies Used

* HTML5
* CSS3
* HTML Forms
* CSS Pseudo-classes
* HTML5 Form Validation
* Responsive Web Design

### Outcome

Successfully designed and implemented a modern, responsive, and accessible **Membership Application Form** with custom styling, interactive user feedback, and built-in HTML5 validation, strengthening my understanding of form design, accessibility, and modern web development practices.
