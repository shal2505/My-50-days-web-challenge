# Day 04 – Reusable Profile Cards

On **Day 4** of the **50 Days Dev Web Development Challenge**, I learned how to build reusable UI components by creating profile cards for the Core Team section. Instead of displaying plain text, I structured each team member's information into a consistent card layout containing a profile image, name, role, and a "View Profile" button.

I used CSS to design visually appealing cards by adding padding, rounded corners, and subtle box shadows. To maintain a clear visual hierarchy, I styled the member names to stand out while keeping the role text smaller and lighter. I also completed the bonus challenge by adding a smooth hover animation that slightly lifts each card and enhances its shadow, creating a modern and interactive user experience.

### What I Learned

* Creating reusable UI components using HTML and CSS.
* Organizing multiple cards with a flexible layout using Flexbox.
* Styling components with `padding`, `border-radius`, and `box-shadow`.
* Building a clear visual hierarchy through typography and spacing.
* Designing interactive cards using the `transition` property.
* Applying `transform: translateY()` and enhanced `box-shadow` effects for smooth hover animations.
* Maintaining consistent styling across multiple reusable profile cards.

### Technologies Used

* HTML5
* CSS3
* CSS Flexbox

### Outcome

Successfully designed a responsive and reusable Core Team section featuring uniform profile cards with modern styling and interactive hover effects. This improved my understanding of component-based design, CSS styling, and creating engaging user interfaces for web applications.
